# Demo Capture Guide

## Goal

Capture the project in a way that is credible for sharing with collaborators, testers, or financiers.

## Recommended Screens

### 1. Hero + Beta Readiness

- show the landing hero;
- show presets loaded;
- include the beta-readiness section in frame when possible.

### 2. Builder Flow

- capture the interactive builder with a realistic preset loaded;
- make sure selections look intentional, not half-filled.

### 3. Sheet Preview

- capture the character sheet preview with stats, features, and spells visible;
- prefer a clean preset with recognizable fantasy identity.

### 4. Preflight State

- capture one clean export state;
- capture one warning state if it is easy to reproduce without noise.

### 5. Foundry Result

- capture the final actor created in Foundry after successful import;
- if possible, capture both the success message and the resulting actor sheet.

## Visual Rules

- prefer one preset that reads clearly in a screenshot;
- avoid noisy technical JSON unless the audience is specifically technical;
- keep the viewport wide enough that the layout feels intentional;
- avoid showing incomplete draft data or debug-only artifacts.

## Minimum Capture Set

- one web hero/demo screenshot;
- one builder screenshot;
- one sheet preview screenshot;
- one Foundry success screenshot.
