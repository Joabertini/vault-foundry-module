Foundry Import Handoff

Reporte tecnico para continuidad de MVP

Contexto

Este reporte deja el estado real del bug de MVP al cierre de la jornada. Los dos sintomas reportados por usuario siguen siendo la ausencia de datos visibles en spells dentro de la UI y la falla de import del JSON exportado a Foundry.

Que se verifico

  • Se leyo el documento tecnico Bertinis-Foundry-Technical-Reference.docx y se uso como guia.
  • La API local ya devuelve metadata rica para spells comunes.
  • El catalogo local ya contiene spells MVP enriquecidos.
  • Se compararon rambo.foundry-actor.json, rambo.foundry-actor2.json y fvtt-Actor-alpaca-L8nBWSEkSrHKABu3.json.

Hallazgos tecnicos

La UI puede seguir mostrando Sin dato si la vista activa depende solo del payload resumido de la grilla. Para bajar ese riesgo, la app ahora resuelve metadata por spellId mezclando fallback local, builder options y detalle puntual de /spells/:id.

En paralelo, el exporter se acerco mas al shape nativo de actor exportado por Foundry: actor root con _id y sort, AC con calc: "default" y flat: null, HP mas completo, movement/senses mas compatibles y slots de spell con override: null.

Siguiente paso

  1. Regenerar un actor nuevo desde la web con este commit.
  2. Intentar importarlo otra vez en Foundry.
  3. Si falla, capturar mensaje exacto de error o consola.
  4. Si el root ya pasa, auditar uno por uno los items embebidos.
